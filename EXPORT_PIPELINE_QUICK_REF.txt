KEY FEATURES ALREADY IMPLEMENTED:
================================================================================

✅ Image Rendering in PDFs:
   - HTTP URLs: Supported (marked token.href.startsWith('http'))
   - data:// URIs: Supported (marked token.href.startsWith('data:'))
   - Example: <Image src={token.href} style={styles.mapImage} />
   - Placement: Anywhere in markdown via ![alt](url) syntax

✅ Link Extraction:
   - Current: Links stripped from text → renderInlineText() line 18
   - Markdown: [text](url) → extracts "text" only
   - Issue: URL discarded; not preserved as clickable element

✅ Route Endpoints Available:
   - /routes/export/google-maps (POST) → Google Maps URL
   - /routes/mapbox/multi-waypoint (POST) → Route geometry + distance/duration
   - /routes/ors (POST) → OpenRouteService routing
   - /routes/google-maps (POST) → Google Maps Routes API

✅ Parallel Processing:
   - All map images fetched in parallel: Promise.all(mapImagePromises)
   - Scales to many destinations efficiently

✅ Content Format:
   - Stored as plain markdown in Note.content field
   - No special formatting required
   - Can include markdown images: ![map](data:image/png;base64,...)

================================================================================
CONSTRAINTS & LIMITATIONS:
================================================================================

⚠️  Clickable Links in PDF:
   - react-pdf/renderer v2 does NOT support <Link> component
   - Workaround: Include URL as plain text or QR code image
   - URL can be printed for manual copying or scanned with QR reader

⚠️  Markdown Parser Complexity:
   - Uses marked.lexer() - splits on line boundaries
   - Custom blocks must be markdown-compatible or HTML-compatible
   - Nested structures limited to token types listed above

⚠️  Font Support (react-pdf built-in):
   - Only standard fonts: Helvetica, Courier, Times-Roman
   - Bold/Italic via fontFamily suffix: "Helvetica-Bold", "Helvetica-Oblique"
   - No custom fonts without additional setup

⚠️  Image Fetch Failures:
   - Currently no timeout; could hang if Mapbox API down
   - Recommendation: Implement 3-5 second timeout + graceful fallback

SAFETY CONSIDERATIONS:
================================================================================

✅ Image Security:
   - Only HTTP/HTTPS URLs allowed (no file://)
   - data:// URIs supported for base64 images
   - Mapbox URLs are external, publicly accessible
   - No image validation needed (PDF library handles)

✅ Link Security:
   - Links included as plain text (no JS execution)
   - Google Maps URLs are safe (no user input in URL)
   - QR codes are just images

⚠️ Markdown Input:
   - Uses DOMPurify for HTML sanitization in MarkdownEditorPanel
   - marked.lexer() is safe (no eval)
   - Recommendation: Sanitize user markdown if allowing [ROUTE_BOX] syntax

⚠️ API Key Management:
   - Mapbox token passed from context (already secured)
   - Google Maps export endpoint public (no key needed)
   - Recommendation: Verify trip_id authorization if fetching routes
